## 配置文件要点（docker-compose.yml）

- `new-api` 仅监听 `127.0.0.1:3000`，**不对公网暴露**，由 Nginx 反代。
- 环境变量：`SQL_DSN`(postgresql) / `REDIS_CONN_STRING`(带密码) / `SESSION_SECRET` / `TZ=Asia/Shanghai` / `ERROR_LOG_ENABLED=true` / `BATCH_UPDATE_ENABLED=true` / `STREAMING_TIMEOUT=300`。
- postgres / redis 均配置 healthcheck 与 `restart: always`。

---

## ⚠️ 已知问题：容器 healthcheck 误报 unhealthy（不影响服务）

| 现象 | 原因 | 处理建议 |
|------|------|---------|
| `docker ps` 中 new-api 显示 `unhealthy` | 脚本生成 healthcheck 用 `curl`，但 `calciumion/new-api` 镜像**仅含 wget**，无 curl | 服务实际正常（API 返回 success）。如需消除误报，改 compose：`wget -q -O - http://localhost:3000/api/status \|\| exit 1`，再 `docker compose up -d` |

> 验证方式：`docker exec new-api wget -qO- http://localhost:3000/api/status` 返回 JSON 即健康。

## Nginx 配置要点

- 同时监听 `80` / `443 ssl` / `443 quic`（HTTP/3，nginx 已编译 http_v3_module）。
- HTTP 80 自动 301 跳转 HTTPS（`.well-known/acme-challenge/` 除外）。
- `location /` → `proxy_pass http://127.0.0.1:3000`。
- **SSE 流式支持**：`proxy_buffering off` / `proxy_cache off` / `proxy_read_timeout 3600` / `chunked_transfer_encoding on`。
- WebSocket 支持：`Upgrade` / `Connection` 头已透传。
- `client_max_body_size 50m`。

## SSL 证书

签发方式：Let's Encrypt（通过 acme.sh，ECC-256，webroot 验证）

- 域名：612.hybaliez.xyz
- 到期：2026-09-12
- 自动续期：已启用（acme.sh cron + `--reloadcmd systemctl reload nginx`）

```bash
# 查看到期时间
openssl x509 -in /usr/local/nginx/conf/ssl/612.hybaliez.xyz/fullchain.pem -noout -dates

# 手动续期（需 root）
~/.acme.sh/acme.sh --renew -d 612.hybaliez.xyz --ecc --force
systemctl reload nginx
```

## 常用命令

```bash
# 进入服务目录
cd /opt/docker-services/new-api

# 容器状态
docker compose ps

# 查看日志
docker compose logs -f new-api

# 重启 / 停止 / 启动
docker compose restart
docker compose stop
docker compose start

# 升级 new-api
cd /opt/docker-services/new-api
docker compose pull
docker compose up -d

# 备份 PostgreSQL
docker compose exec postgres pg_dump -U newapi newapi > backup_$(date +%Y%m%d).sql

# Nginx 测试与重载
/usr/local/nginx/sbin/nginx -t
systemctl reload nginx

# 健康检查（公网）
curl -s https://612.hybaliez.xyz/api/status | head -c 200
```
